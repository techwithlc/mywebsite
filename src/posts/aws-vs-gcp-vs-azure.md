After spending 2+ years as an AWS Cloud Support Engineer in Dublin and building side projects on GCP and Azure, I get this question constantly: **which cloud should I learn first?**

Short answer: AWS. Long answer: it depends — but still probably AWS.

## Why AWS Wins for Beginners

The sheer volume of learning resources, community, and job listings for AWS is unmatched. When I joined AWS Dublin in 2022, I was handling 500+ enterprise customers across all industries. Almost all of them were primarily on AWS.

- **Job market**: AWS certifications (especially SAA and SA Pro) are the most recognized globally
- **Community**: r/aws, AWS re:Invent, local user groups — the ecosystem is massive
- **Documentation**: Arguably the best of the three

## Where GCP Shines

If you're going deep on data engineering, ML, or Kubernetes (GKE is genuinely excellent), GCP is worth your time. BigQuery is class-leading for analytics. But the job market is thinner.

## Azure's Niche

Enterprise. If you're targeting Fortune 500 companies running Microsoft stacks — Windows Server, Active Directory, .NET — Azure is the play. AZ-900 is also the easiest cloud cert to get as a starting point.

## My Honest Recommendation

1. **Get AWS SAA first** — it opens the most doors
2. **Learn one other cloud** to the point you can work in it (not certify, just operate)
3. **Certs are table stakes now** — what employers want is real incident experience

The cloud doesn't matter as much as the fundamentals: networking, IAM, observability, and on-call discipline. Those transfer anywhere.
