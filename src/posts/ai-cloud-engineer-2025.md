每次我在 YouTube 講 AI 的話題，留言區一定有人問：「以後還需要學雲端嗎？AI 不會把這些工作都取代嗎？」

我的答案是：**AI 會取代你工作的某些部分，但同時讓你剩下的那部分更值錢。**

## 哪些會被取代

老實說，很多「入門級」的工作確實在消失中：

- 基本的 Terraform 模板撰寫
- 標準化的 CI/CD pipeline 設定
- 照著 runbook 執行的例行維護

這些 AI coding assistant 已經做得很好了。如果你的工作只是這些，確實要開始擔心。

## 哪些反而更值錢

但以下這些，AI 短期內很難取代：

**架構決策能力** — 知道什麼時候用 RDS、什麼時候用 Aurora Serverless、什麼時候根本不應該用關聯式資料庫，這需要真實的系統設計經驗，不是 prompt。

**生產事故處理** — 凌晨三點、cascading failure、客戶在燒錢，你需要的是一個能做決策的人，不是一個 AI 在生成假設。

**跨系統的 observability 思維** — 能看出一個 trace 裡哪個 span 是問題根源，這需要對系統有深度理解。

## 我的建議

1. **把 AI 當工具用，不要當拐杖** — 用 GitHub Copilot、用 Claude，但要能解釋你 accept 的每一行 code
2. **往深度走，不要往廣度走** — 成為某個領域真正的 expert（Kubernetes networking、AWS cost optimization、security response）
3. **培養溝通能力** — 能把技術問題翻譯成商業語言的工程師，永遠搶手

AI 是工具，不是你的替代品。除非你一直只做工具能做的事。
